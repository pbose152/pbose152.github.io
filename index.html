üìã Problem Statement

Excessive startup waste at 3UC Ultracast coating line during production changeovers. Baseline average: 282.7 lbs per startup event with significant variation across operators and product widths.

üéØ Project Goal

Achieve 18% reduction in startup waste with projected annual savings of $76K+. Scope focused on 3UC coater line startup events.

üìä Data Collection

Compiled 339 startup events from production records. Extracted NET_LBS, footage, startup time, operator IDs, and product specifications from PI Vision and Braincube systems.

üìà Baseline Metrics

Baseline average: 282.7 lbs per startup. Analyzed 100,000+ rows of production data using Python (Pandas, NumPy) and Minitab for statistical analysis.

üîç Statistical Analysis

Performed ANOVA testing for shift effects (p=0.895 - NOT significant). Correlation analysis showed strong relationship between NET_LBS and Time (r=0.905). Identified AutoTurnup as key variable.

üí° Root Cause Findings

AutoTurnup ON reduces waste by ~12%. Identified 5 operators for targeted training. Discovered width-specific patterns requiring differentiated targets.

üéØ Width-Specific Targets

Established optimized targets: 56" ‚Üí 186 lbs, 65.5" ‚Üí 198 lbs, 86.5" ‚Üí 265 lbs. Universal targets: 730 ft footage and 6 min startup time.

Linear String Voids (LSV) Event Analysis

Root Cause Analysis & Process Optimization

Problem

Linear String Voids coating defects occurring sporadically across production runs, causing quality issues and customer claims. Needed to identify process parameters differentiating good runs from bad runs.

Approach

  • Compiled 17 documented LSV events from 2023-2025 production records
  • Separated analysis by product type: 30Q (6 events) vs 80Q (11 events)
  • Extracted roll speed data for ¬±10 day windows around events
  • Compared process parameters between bad runs and normal production

Key Findings

  • Greater speed difference than usual between gravure and transfer roll was one of the primary causes
  • Damaged gravure roll identified as another possible root cause
  • Different control strategies needed for 30Q vs 80Q product lines
  • Created comparison framework for identifying anomalous process conditions

Tools Used: Python, PI Vision, Braincube, Excel

Statistical Quality Control Framework

Weekly Executive Reporting System

Problem

Need for systematic monitoring of coating process parameters across three production lines to identify trends, detect out-of-control conditions, and communicate performance to management.

Approach

  • Developed weekly reporting templates for viscosity, coatweight, glycol, silicone
  • Created control charts in Minitab with 15-point rolling windows
  • Established factor adjustment protocols for viscosity deviations
  • Built executive summary format highlighting key variances

Key Findings

  • Viscosity factor adjustments reduced out-of-control events by >50%
  • Coatweight tolerance of ¬±2 points maintained across production
  • Identified systematic E288 formula drift requiring recipe adjustment

Tools Used: Minitab, Excel, PI Vision, Braincube

Operator Performance Analytics

Data-Driven Performance Benchmarking

Problem

Significant variation in startup waste performance across operators. Need to identify best practices from top performers and develop targeted training for operators with higher waste rates.

Approach

  • Analyzed operator performance using NET_LBS and startup time metrics
  • Created individual I-MR control charts for top operators
  • Performed shift-based analysis to isolate operator vs. systemic effects
  • Generated Pareto charts to identify training priorities

Key Findings

  • Top 5 operators account for majority of startups
  • Significant variation in mean NET_LBS across operators (not shifts)
  • Best operators achieve 20%+ lower waste than average

Tools Used: Python, Minitab, Excel

5S Workplace Organization

Lean Manufacturing Implementation

Problem

Need for standardized workplace organization across coating lines to improve operational efficiency, reduce search time for tools/materials, and enhance safety compliance.

Approach

  • Developed 8-week phased implementation plan for 1UC and 4UC
  • Created 5S audit checklist with 27 items across all five S categories
  • Established scoring system (1-5 scale) with 3.5+ pass threshold
  • Designed red-tag process for Sort phase

Key Findings

  • Structured audit checklist improves completion rates
  • Shadow boards reduce tool search time significantly
  • Weekly mini-audits maintain gains better than monthly reviews
  • Operator ownership key to sustaining improvements

Tools Used: Word, Excel
